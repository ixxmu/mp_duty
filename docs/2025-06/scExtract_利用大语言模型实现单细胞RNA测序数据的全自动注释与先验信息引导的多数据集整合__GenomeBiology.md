Genomics平台）的广泛应用，单细胞RNA测序已渗透到各个生物医学研究领域。为减少因重复测序造成的研究资源浪费，第三方公共数据集已成为科研发现与验证不可或缺的资源。因此，针对重大复杂疾病的大规模、经整理的单细胞“图谱”数据集也应运而生。像人类细胞图谱（HCA）这样的合作项目，以及诸如cellxgene这样的众包平台，推动了涵盖多物种、跨组织和多组学的细胞数据集的生成。<span textstyle="">截至2024年8月9日，作为目前最大、基于文献整理的单细胞数据库，cellxgene共收录了1458个数据集，主要为人和小鼠的单细胞RNA测序数据。然而，这一进展被每年大量涌现的、包含新单细胞测序数据的科研论文所掩盖（见图1A）。这些分散的数据集涵盖了从基础细胞过程到多种人类病理状态的研究内容，包括癌症、神经退行性疾病和免疫相关疾病等。对这些资源进行高效利用，有望加深我们对常见病与罕见病的理解，并促进构建一个覆盖广泛组织类型、发育阶段、疾病状态及治疗条件的多层次、多维度细胞图谱，从而提升科研效率，减少重复性工作。</span></span></p><section textstyle="">过滤（Filtering）：该阶段基于质控指标去除低质量细胞。默认使用标准参数，但如果文章中明确提到了额外的过滤条件，则会解析并应用这些条件。</span></span></p></li><li><p><span leaf=""><span textstyle="">预处理（Preprocessing）：为聚类分析做准备，调整或从文章中推断关键参数，如高变基因数量和 kNN 图的邻域大小。</span></span></p></li><li><p><span leaf=""><span textstyle="">聚类（Clustering）：scExtract 自动判断使用的聚类方法（Leiden 或 Louvain），并估计细胞簇的数量，通常通过从文章中提取或推断相关信息，以与作者的生物学理解保持一致。</span></span></p></li><li><p><span leaf=""><span textstyle="">注释（Annotation）：基于每个簇的标记基因列表进行初步注释，并将这一信息与文章中的背景知识结合，确保注释结果与研究内容一致。同时注释的内容包括细胞类型、置信度、组织来源、疾病状态以及发育阶段等。</span></span></p></li><li><p><span leaf=""><span textstyle="">重注释（Re-annotation，可选）：scExtract 可通过自主查询特征标记基因的表达水平（这些基因基于文章内容及已有组织/细胞类型信息生成），对置信度较低的细胞群进行优化，从而提升注释准确性。</span></span></p></li><li><p><span leaf=""><span textstyle="">在数据整合阶段，scExtract 利用了前期生成的注释信息，并对两个表现优异的软件 scanorama 和 cellhint 进行了改进，分别形成了 scanorama-prior 和 cellhint-prior 方法：</span></span></p></li><li><p><span leaf=""><span textstyle="">scanorama-prior：该方法引入了来自 scExtract 流程中的聚类信息和细胞注释相似性矩阵。在构建互近邻（MNNs）时，考虑不同细胞类型之间的先验差异，调整加权距离以更准确地构建邻居关系。此外，还引入了一个基于细胞群中心的额外调整向量，在保留生物关系的前提下增强批次效应校正效果，使相似细胞群更加一致地被整合。</span></span></p></li><li><p><span leaf=""><span textstyle="">cellhint-prior：专为处理粗略注释设计，cellhint-prior 在整合过程中根据细胞群体分配的不确定性动态调整先验知识的权重。它能够自动协调不同数据集间的注释差异，通过跨数据集比较生成统一的注释结果。</span></span></p></li><li><p><span leaf=""><span textstyle="">scExtract 实现了一个两步整合流程：首先使用 cellhint-prior 进行细胞类型层面的注释协调，利用相邻数据集的信息；随后使用 scanorama-prior 对协调后的细胞类型进行嵌入空间层面的整合。该流程同时解决了异构单细胞组学数据集中注释不一致与嵌入整合的双重挑战。</span></span></p></li></ul><p><span leaf="">本研究主要采用 Claude 3.5 Sonnet、Deepseek-v2.5 和 GPT-4o-mini 作为大语言模型服务提供方，选择它们是因其具备较大的上下文窗口和支持大规模查询的性价比优势。</span></p><p><span leaf=""><span textstyle="">分析方法与评估 </span></span></p><p><span leaf="">本研究从多个方面对 scExtract 进行了系统评估：</span></p><p><span leaf=""><span textstyle="">1. 聚类与注释准确性：数据集：使用来自 cellxgene 的 18 个中等规模手动注释数据集（每个约含 10^4 个细胞）作为基准。对比方法：将 scExtract 与 SingleR、scType 和 CellTypist 等成熟的参考图谱迁移方法进行比较。 评估指标： 使用调整兰德指数（ARI）评估聚类性能和群体结构的保留情况。利用文本到嵌入的方法将注释转换为向量表示，通过余弦相似度评估注释准确性。基于细胞本体标签的相似性评估，采用层级节点间的 Jaccard 系数计算。先验知识的影响：比较了 scExtract 在引入背景知识与否的情况下的表现，并与 GPTCelltype 进行对比。 置信度评分：评估 scExtract 所生成的注释置信度是否与实际注释准确性相关。</span></span></p><p><span leaf=""><span textstyle="">2. 输出稳定性与提示敏感性：方法：对选定样本重复分析六次，以评估参数提取和细胞注释的可重复性。提示变化测试：设计五种不同的提示变体（指令重排序、术语修改、增强引导、极简注释、结构重构），评估其对性能的影响。</span></span></p><p><span leaf=""><span textstyle="">3. 整合性能：数据集：使用不同测序平台获取的胰腺单细胞 RNA 测序数据集进行基准测试。 评估指标：通过校正后的 PCA 和 UMAP 嵌入评估批次效应去除能力和生物多样性保留程度。对注释错误的鲁棒性：在三种错误标注场景下测试整合性能：命名相近、无偏误错误命名、有偏误错误命名。大规模数据集测试：在六个包含 5 万至 60 万个细胞的大规模数据集上评估 scExtract 的两步整合策略，模拟典型的实际应用场景。运行时间性能：在普通 PC 处理器和 GPU 加速服务器上分别评估其计算效率和扩展性。</span></span></p><p><span leaf=""><span textstyle="">4. 皮肤自身免疫疾病数据集的自动构建：应用：利用 scExtract 整合 14 个涵盖多种皮肤状态（银屑病、特应性皮炎、痤疮、环状肉芽肿、发育阶段）的皮肤单细胞 RNA-seq 数据集，构建了一个包含超过 44 测序数据注释与整合的全流程自动化，显著减少了繁琐的手动预处理工作，提高了研究效率。该框架通过大语言模型从科研文献中提取实验参数和生物学知识，简化了分析流程，使研究人员能够更专注于生物学意义的挖掘。其在注释与整合过程中融合先验知识的能力，配合出色的鲁棒性与可扩展性，使其成为单细胞组学领域的重要基础设施。它降低了大规模单细胞分析的门槛，加速了组织特异性图谱的构建，并推动了对稀有细胞类型和复杂疾病机制的新发现。</span></span></p><p><span leaf="">新开发或优化的生物计算方法：</span></p><p><span leaf=""><span textstyle="">1. scExtract 框架：总体目标：scExtract 是一个基于大语言模型（LLMs）的全新综合性框架，用于实现单细胞 RNA 测序（scRNA-seq）数据的全自动注释与先验信息引导下的多数据集整合。它通过自动从科研文献中提取实验参数和生物学知识，解决了处理海量公共数据集时所面临的关键瓶颈。 输入数据：仅需原始表达矩阵（以 h5ad 格式）和对应科研文章内容（PDF 格式）作为输入。工作流程：该流程模拟专家分析步骤，包含四个主要阶段：过滤、预处理、聚类以及两轮注释，随后进行整合。 过滤（Filtering）：去除质量控制（QC）指标不佳的细胞。应用了每个细胞的基因数、总 UMI 数及表达基因数量的阈值。默认参数（例如 10X 数据中每个细胞至少 300 个基因）已预设，而额外的过滤条件（如线粒体/核糖体基因比例）仅在文章中明确提及并被 LLM 解析后才会启用。</span></span></p><p><span leaf=""><span textstyle="">预处理（Preprocessing）：为聚类分析做准备。关键可调参数包括高变基因数量、kNN 图构建中的邻域大小，也可选择使用 harmonypy 进行样本层面的批次校正。 </span></span></p><p><span leaf=""><span textstyle="">聚类（Clustering）：决定使用的聚类方法（Leiden 或 Louvain）以及估计的细胞簇数量。scExtract 根据文章内容推断出的簇数量或作者明确说明的信息，选取合适的分辨率参数，从而更贴合生物学意义。  </span></span></p><p><span leaf=""><span textstyle="">注释（Annotation）：利用 LLM 智能体执行注释任务。以每个细胞群通过 Wilcoxon 秩和检验识别出的标记基因为输入，并结合文章背景知识进行综合判断。注释内容涵盖细胞类型、置信度、组织来源、疾病状态以及发育阶段等信息。 </span></span></p><p><span leaf=""><span textstyle="">重注释（Re-annotation，可选）：在初步注释完成后，scExtract 可基于文章内容和已有组织/细胞类型信息自主生成特征标记基因集合，并查询其表达水平对低置信度的注释结果进行优化。  输出与文档记录：除处理后的数据集外，scExtract 还生成配置文件记录所有参数，并输出详细的日志文件，说明参数选择与注释背后的推理过程，便于用户验证和手动修正。 </span></span></p><p><span leaf=""><span textstyle="">大语言模型集成：采用多个支持大规模查询、具备长上下文窗口和合理价格的大语言模型服务提供方，如 Deepseek-v2.5、GPT-4o-mini 和 Claude-3.5-sonnet。其中，Claude 3.5 Sonnet 在信息提取准确性方面表现最佳，且对提示词变化的敏感性较低。</span></span></p><p><span leaf=""><span textstyle="">自我反思机制：其出色的注释能力源于结合了来自文章的先验知识以及通过两轮注释实现的自我反思能力，即对自身输出进行回顾与调整。</span></span></p><p><span leaf=""><span textstyle="">稳定性与鲁棒性：在多次重复运行和不同提示设置下，scExtract 表现出良好的稳健性，尤其在参数和细胞类型定义清晰的情况下。对于需要细致解读模糊信息的情境，性能波动也能够通过模型自述的置信度准确反映出来。</span></span></p><p><span leaf=""><span textstyle="">2. scanorama-prior：</span></span></p><p><span leaf=""><span textstyle="">目的：这是对 scanorama 方法的改进版本，通过引入先验注释信息来提升多数据集整合过程中的批次效应校正能力，并在整合过程中更好地保留生物多样性。它解决了传统整合方法无法利用已有注释信息的局限性。</span></span></p><p><span leaf=""><span textstyle="">机制： 互近邻（MNN）构建：在构建互近邻时，scanorama-prior 会考虑不同细胞类型之间的先验差异。它根据命名相似性调整跨数据集细胞间的加权距离，从而实现更准确的邻居构建。  整体移动策略：倾向于将原始细胞群作为一个整体向目标数据集中对应的细胞群移动，从而增强批次校正效果。附加调整向量：基于两个数据集中细胞群中心的位置，施加一个额外的调整向量。该向量的权重由原始组与目标组之间的注释相似性决定。</span></span></p><p><span leaf=""><span textstyle="">公式： MNN 的修正距离：d′ij = dij / MIJ；其中 d′ij 表示细胞 i 和 j 之间的修正距离；dij 是细胞 i 和 j 之间的原始距离； MIJ 表示细胞类型 I 和 J 之间的相似性。 偏差调整公式：Bias′ij = Biasij + MIJ × (vi − vI − vj + vJ )；Bias′ij 是调整后的偏差，Biasij 是细胞 i 和 j 之间的原始偏差；MIJ 是细胞类型 I 和 J 之间的相似性；vi 是细胞 i 的嵌入向量；vI 是细胞类型 I 的平均嵌入向量；vj 是目标细胞 j 的嵌入向量；vJ 是目标细胞类型 J 的平均嵌入向量。 参数与细节：需要 scExtract 流程中提供的聚类信息以及细胞注释的相似性矩阵。为了加快 kNN 图的构建，候选细胞池扩展为 k 值的五倍（可配置）。在 V100 GPU 上运行时，表现出良好的可扩展性，能处理高达一百万个细胞的数据集，batch_size 参数设置为 1000 个细胞以避免内存溢出。敏感性：尽管效果显著，但 scanorama-prior 对注释错误较为敏感，特别是在存在有偏误的错误命名时。</span></span></p><p><span leaf=""><span textstyle="">3. cellhint-prior： 目的：这是 cellhint 的一种融合先验知识的改进版本，旨在自动协调不同数据集之间的注释差异，尤其用于弥补 scanorama-prior 对注释错误的敏感性问题，适用于初步或粗略注释的场景。 机制：动态权重调整：根据细胞匹配不确定性程度动态调整注释权重。当不确定性较高时，cellhint-prior 会更多地依赖先验知识。相似度矩阵归一化：将细胞与细胞类型之间的原始距离矩阵进行归一化，并转化为相似度矩阵。公式：调整后相似度：S′iJ = β[(1− αi)SiJ + αiMIJ ] + (1− β)SiJ；S′iJ 表示细胞 i 与目标细胞类型 J 的调整后相似度；β 是控制先验信息影响的超参数（默认值为 0.1）；αi 是清晰度评分的倒数，基于细胞 i 到所有细胞类型的熵计算得出；SiJ 是细胞 i 与目标细胞类型 J 的原始相似度；MIJ 是细胞类型 I 与 J 之间的相似度。</span></span></p><p><span leaf=""><span textstyle="">清晰度评分：用于衡量细胞表达谱的区分度。当清晰度下降（表示难以区分细胞类型）时，先验知识的影响更加显著。</span></span></p><p><span leaf=""><span textstyle="">鲁棒性：在面对注释变化和命名误差时表现出更强的稳定性与适应能力。</span></span></p><p><span leaf=""><span textstyle="">两步自动化整合流程： 流程：scExtract 实施了一个分阶段的整合策略：首先使用 cellhint-prior 进行细胞类型层面的注释协调，以纠正可能存在的命名不一致问题；随后再使用 scanorama-prior 对协调后的细胞类型进行嵌入空间层面的整合。
