Cell Systems｜BulkFormer 之基于 bulk RNA-seq 数据的大模型来了 by 生信宝库
------
<div><section data-pm-slice="0 0 []"><section><span><span leaf="">生信文献解读</span></span></section><h2><span leaf="">写在前面</span></h2><p><span leaf="">过去两年，Geneformer、scGPT、scFoundation 等转录组基础模型快速发展，但它们大多在单细胞 RNA-seq 上预训练。问题在于，单细胞矩阵的稀疏性、基因覆盖和样本单位，与临床队列中常见的 bulk RNA-seq 并不相同。把单细胞模型直接迁移到 bulk 数据，未必是最合适的技术路线。</span></p><p><span leaf="">2026 年，北京大学崔庆华团队在 Cell Systems 发表 </span><strong><span leaf="">BulkFormer: A large-scale foundation model for bulk transcriptomes</span></strong><span leaf="">，提出专门面向人类 bulk 转录组的基础模型 BulkFormer。模型覆盖 20,010 个蛋白编码基因，约 1.5 亿参数，在 581,503 个人类 bulk RNA-seq 表达谱上完成预训练。</span></p><section nodeleaf=""><img data-aistatus="1" data-imgfileid="100015272" data-s="300,640" data-src="https://mmbiz.qpic.cn/sz_mmbiz_png/I7BPRRicNMlRBqbNNTNsSprOibfXKuqJiaDtVa475xibmiczicax1gpl8WIE5j22RSgUFYXUujItJzpm05RIEUAvMlK7kjFLUQjKj7KXwZJ3dkdlY/640?wx_fmt=png&amp;from=appmsg" data-type="png" type="block" src="https://mmbiz.qpic.cn/sz_mmbiz_png/I7BPRRicNMlRBqbNNTNsSprOibfXKuqJiaDtVa475xibmiczicax1gpl8WIE5j22RSgUFYXUujItJzpm05RIEUAvMlK7kjFLUQjKj7KXwZJ3dkdlY/640?wx_fmt=png&amp;from=appmsg"></section><p><span leaf=""><br></span></p><p><span leaf="">这篇文章最值得关注的，不只是又多了一个转录组大模型，而是它系统回答了一个更基础的问题：</span><strong><span leaf="">对于下游 bulk RNA-seq 任务，模型结构重要，预训练数据与目标任务的模态匹配同样重要。</span></strong></p><h2><span leaf="">一、为什么 bulk RNA-seq 需要自己的基础模型</span></h2><p><span leaf="">bulk RNA-seq 与单细胞 RNA-seq 都测量基因表达，但两者不是简单的“样本数不同”。</span></p><p><strong><span leaf="">单细胞数据的典型特点</span></strong></p><ul><li><section><span leaf="">一个细胞通常只能检测到约 3,000 个基因；</span></section></li><li><section><span leaf="">dropout 和零值很多，矩阵高度稀疏；</span></section></li><li><section><span leaf="">每个 token 或样本单位对应单个细胞；</span></section></li><li><section><span leaf="">细胞状态和细胞类型差异占据主要变化来源。</span></section></li></ul><p><strong><span leaf="">bulk 数据的典型特点</span></strong></p><ul><li><section><span leaf="">一个样本可覆盖约 16,000 个基因；</span></section></li><li><section><span leaf="">表达矩阵更稠密，连续表达值的信息更完整；</span></section></li><li><section><span leaf="">一个样本是多细胞类型和多状态的混合结果；</span></section></li><li><section><span leaf="">常与诊断、分型、预后、药物反应等临床标签配对。</span></section></li></ul><p><span leaf="">因此，用稀疏单细胞数据学到的表达表示，并不一定能完整描述 bulk 样本中的全局表达依赖。BulkFormer 的出发点，就是让预训练数据、基因空间和下游任务处于同一种数据模态中。</span></p><h2><span leaf="">二、预训练数据和模型规模</span></h2><p><span leaf="">BulkFormer 使用 581,503 个人类 bulk RNA-seq 表达谱进行预训练，词表包含 20,010 个蛋白编码基因。作者发布了五个规模的模型，方便在性能与计算成本之间选择。</span></p><p><strong><span leaf="">BulkFormer-37M</span></strong></p><p><span leaf="">约 3,700 万参数，隐藏维度 128，1 层 Performer。适合先跑通流程或做资源受限的小规模测试。</span></p><p><strong><span leaf="">BulkFormer-50M</span></strong></p><p><span leaf="">约 5,000 万参数，隐藏维度 256，2 层 Performer。适合快速比较不同特征和分类器。</span></p><p><strong><span leaf="">BulkFormer-93M</span></strong></p><p><span leaf="">约 9,300 万参数，隐藏维度 512，6 层 Performer。性能和资源开销相对均衡。</span></p><p><strong><span leaf="">BulkFormer-127M / 147M</span></strong></p><p><span leaf="">隐藏维度均为 640，分别为 8 层和 12 层 Performer。147M 是当前仓库默认配置，也是论文和示例中重点使用的最大模型。</span></p><p><span leaf="">所有版本均使用 8 个全局注意力头。仓库还提供 scaling law 拟合图，说明随着模型规模增加，性能呈现可预测的改善趋势。</span></p><section data-src="https://mmbiz.qpic.cn/sz_mmbiz_png/I7BPRRicNMlRtELniaZMlLJNyRoQxXgPPqdibSyK76Uw3EUYibd0Z5EkId1BA6eOQ4gTOYkb8dicR5M8lLdbcjH38u050zyUQVNbwYZSjibPU1Jjc/640?wx_fmt=png&amp;from=appmsg" data-type="png" type="block" src="https://mmbiz.qpic.cn/sz_mmbiz_png/I7BPRRicNMlRtELniaZMlLJNyRoQxXgPPqdibSyK76Uw3EUYibd0Z5EkId1BA6eOQ4gTOYkb8dicR5M8lLdbcjH38u050zyUQVNbwYZSjibPU1Jjc/640?wx_fmt=png&amp;from=appmsg"></section><h2><span leaf="">三、核心架构：GNN 加 Performer</span></h2><p><span leaf="">BulkFormer 的编码器不是单一 Transformer，而是一个混合结构。</span></p><p><strong><span leaf="">第一部分：图神经网络捕捉显式基因关系</span></strong></p><p><span leaf="">作者将基因关系组织成图结构，让模型能够利用已知或数据驱动的 gene-gene connection。这样做的意义是把基因之间的局部关系直接写进模型，而不是完全依赖注意力机制从头学习。</span></p><p><strong><span leaf="">第二部分：Performer 建模全局表达依赖</span></strong></p><p><span leaf="">一个 bulk 样本包含约 2 万个基因。标准自注意力的计算和显存开销会随序列长度快速增加，Performer 用更高效的近似注意力处理长序列，负责捕捉跨基因的全局依赖。</span></p><p><strong><span leaf="">第三部分：表达值、基因身份和样本上下文共同编码</span></strong></p><p><span leaf="">当前代码把表达嵌入、基因身份嵌入和样本级表达上下文相加，再进入图模块和 Performer。最新版还加入三个全局统计量：基因缺失比例、观测基因平均表达、非零基因比例，用于增强对不同队列基因覆盖和稀疏度差异的适应能力。</span></p><p><span leaf="">因此，BulkFormer 学到的不是一张静态基因向量表，而是</span><strong><span leaf="">随样本表达背景变化的基因表示</span></strong><span leaf="">。同一个基因在不同患者样本中，可以拥有不同的上下文 embedding。</span></p><h2><span leaf="">四、五类下游任务表现</span></h2><p><span leaf="">作者在五类任务中比较了 BulkFormer 与多种单细胞基础模型。</span></p><p><strong><span leaf="">1. 转录组表达补全</span></strong></p><p><span leaf="">BulkFormer 的表达补全 PCC 达到 0.954。这个任务与其 masked-expression 预训练目标高度一致，也是 bulk 模态预训练优势最直接的体现。</span></p><p><strong><span leaf="">2. 疾病和组织分类</span></strong></p><p><span leaf="">在 DiSignAtlas、心血管疾病、TCGA 和 GTEx 等数据上，BulkFormer 的 weighted F1 分别达到 0.949、0.978、0.907 和 0.969。这里更重要的不是单个数字，而是模型在不同疾病与组织来源中都保持了较稳定的迁移能力。</span></p><p><strong><span leaf="">3. 预后建模</span></strong></p><p><span leaf="">基础模型先将高维表达矩阵转为低维表征，再连接生存分析模型。对于样本量有限、基因维度很高的临床队列，这种“预训练表示加轻量下游模型”的路线具有现实意义。</span></p><p><strong><span leaf="">4. 药物反应预测</span></strong></p><p><span leaf="">在细胞系 bulk RNA-seq 药敏任务中，BulkFormer 的平均 PCC 为 0.373，高于论文所比较的单细胞基础模型。需要注意，药敏关联仍然受到细胞系与患者之间的 domain shift 影响。</span></p><p><strong><span leaf="">5. 基因必需性预测</span></strong></p><p><span leaf="">在 gene essentiality 任务中，BulkFormer 的平均 PCC 为 0.186。该任务数值整体不高，说明从基础表达推断基因依赖仍然是一个困难问题，但 bulk 预训练表示提供了可用增益。</span></p><section nodeleaf=""><img data-aistatus="1" data-imgfileid="100015275" data-s="300,640" leaf="">五、这项工作的主要创新点</span></h2><p><strong><span leaf="">创新一：预训练数据与目标模态一致</span></strong></p><p><span leaf="">BulkFormer 不是把单细胞模型换个下游数据集，而是从预训练阶段就使用大规模人类 bulk RNA-seq。这是文章最核心的设计判断。</span></p><p><strong><span leaf="">创新二：显式基因图与全局依赖联合建模</span></strong></p><p><span leaf="">GNN 负责局部、显式的基因关系，Performer 负责长序列的全局表达依赖，两类归纳偏置互补。</span></p><p><strong><span leaf="">创新三：统一支持样本级、基因级和表达级输出</span></strong></p><p><span leaf="">当前代码既可以聚合得到样本 embedding，也可以保留每个样本中的基因 embedding，还可以输出缺失基因的预测表达。这让模型可以连接分类、生存、回归和插补等多类分析。</span></p><p><strong><span leaf="">创新四：用同一模型覆盖多个临床相关任务</span></strong></p><p><span leaf="">疾病注释、预后、药敏和基因必需性不再分别从原始表达矩阵训练完全独立的特征提取器，而是共享预训练表示。</span></p><h2><span leaf="">六、如何正确理解结果</span></h2><p><span leaf="">首先，BulkFormer 的结果支持“模态匹配很重要”，但不能简单推导为“bulk 模型在所有任务上一定优于单细胞模型”。如果问题本身是细胞类型注释、细胞轨迹或稀有亚群识别，单细胞模型仍然更合适。</span></p><p><span leaf="">其次，BulkFormer 不是去卷积工具。它可以学习混合组织的表达表示，但不会自动告诉我们每种细胞的比例。需要细胞组成时，仍应结合 CIBERSORTx、MuSiC、BayesPrism 或单细胞参考图谱。</span></p><p><span leaf="">第三，基础模型 embedding 不能替代差异表达、批次评估和独立验证。一个性能更好的分类器，也可能学习到队列来源、建库方案或组织纯度等混杂因素。</span></p><p><span leaf="">第四，仓库已删除旧版的 LINCS 化合物扰动预测任务。作者明确指出 LINCS 表达谱并非 bulk RNA-seq，测量平台和基因覆盖不同，会造成模态不匹配。后续使用时不要照搬旧版本代码或旧结果。</span></p><h2><span leaf="">七、小结</span></h2><p><span leaf="">BulkFormer 的价值可以概括为一句话：</span><strong><span leaf="">它让 bulk RNA-seq 第一次拥有了与自身数据结构相匹配的大规模基础模型。</span></strong></p><p><span leaf="">对常规生信项目来说，短期内最实用的用法不是重新预训练 1.5 亿参数模型，而是下载作者权重，提取样本 embedding，再接入疾病分型、预后或药敏模型。下一篇我们会从原始 count 矩阵开始，完整跑一遍最新版特征提取流程。</span></p><h2><span leaf="">参考资料</span></h2><p><span leaf="">1.
