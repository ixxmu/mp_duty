<div><section data-tool="mdnice编辑器" data-website="https://www.mdnice.com" data-pm-slice="0 0 []"><h2 data-cacheurl="" data-remoteid="" data-tool="mdnice编辑器"><span><span leaf="">为什么要进行伪时序分析？</span></span></h2><p data-tool="mdnice编辑器"><span leaf="">在常规单细胞 RNA-seq 实验中，每个细胞通常只在一个时间点被测序。测序本身会破坏细胞，因此不能像活细胞成像那样直接追踪同一个细胞从早期状态到晚期状态的真实轨迹。</span></p><p data-tool="mdnice编辑器"><span leaf="">但是，很多生物过程并不是同步发生的。例如造血分化、胚胎发育、上皮细胞成熟、免疫细胞激活、肿瘤细胞状态转变等，同一个样本中可能同时捕获到处在不同进程阶段的细胞。伪时序分析（pseudotemporal ordering）的目的，就是利用这些 snapshot 数据中的转录组连续变化，把细胞按照某个生物过程的相对进展顺序排列起来。</span></p><p data-tool="mdnice编辑器"><span leaf="">简单理解就是：</span></p><pre data-tool="mdnice编辑器"><code><span leaf="">离散捕获的单细胞状态 → 推断连续变化过程 → 给每个细胞一个相对顺序</span><br></code></pre><p data-tool="mdnice编辑器"><span leaf="">这个相对顺序称为 pseudotime。在 root cell 选择合理、轨迹结构成立的前提下，pseudotime 小的细胞通常被解释为更接近起始状态，pseudotime 大的细胞通常被解释为更接近终末状态。</span></p><p data-tool="mdnice编辑器"><span leaf="">需要注意的是，伪时序不是实际时间。它的单位不是小时、天或发育阶段，而是细胞在表达空间中沿着某条推断轨迹前进的相对位置。因此，伪时序适合回答：</span></p><ul><li><section><span leaf="">某个细胞群内部是否存在连续状态变化。</span></section></li><li><section><span leaf="">某些 marker gene 是否沿着状态转换逐渐上升或下降。</span></section></li><li><section><span leaf="">一个起始状态是否可能分叉到多个终末状态。</span></section></li><li><section><span leaf="">不同样本或条件中的细胞是否分布在不同的伪时序阶段。</span></section></li></ul><p data-tool="mdnice编辑器"><span leaf="">它不适合直接回答：</span></p><ul><li><section><span leaf="">一个细胞真实经过了多少小时才到达终末状态。</span></section></li><li><section><span leaf="">某两个细胞之间是否存在真实祖先-后代关系。</span></section></li><li><section><span leaf="">某个基因变化是否一定是导致分化的因果因素。</span></section></li><li><section><span leaf="">任意两个 UMAP 上相邻的 cluster 是否存在发育关系。</span></section></li></ul><h2 data-cacheurl="" data-remoteid="" data-tool="mdnice编辑器"><span><span leaf="">伪时序分析适合什么数据？</span></span></h2><p data-tool="mdnice编辑器"><span leaf="">伪时序分析依赖一个关键假设：数据中确实存在一个主要的、连续的生物过程，并且这个过程能够在转录组空间中被捕捉到。</span></p><p data-tool="mdnice编辑器"><span leaf="">适合做伪时序的常见场景包括：</span></p><ul><li><section><span leaf="">发育或分化过程，例如 HSC 到髓系或红系细胞。</span></section></li><li><section><span leaf="">组织成熟过程，例如上皮细胞从 progenitor 到 mature cell。</span></section></li><li><section><span leaf="">免疫激活或耗竭过程，例如 naive T cell 到 effector/exhausted T cell。</span></section></li><li><section><span leaf="">肿瘤细胞状态连续变化，例如 EMT、增殖、缺氧或治疗反应状态。</span></section></li><li><section><span leaf="">时间序列实验中同一谱系的多个时间点。</span></section></li></ul><p data-tool="mdnice编辑器"><span leaf="">不太适合直接做伪时序的情况包括：</span></p><ul><li><section><span leaf="">样本中主要是彼此无关的成熟细胞类型，例如普通 PBMC 中把 T cells、B cells、monocytes、NK cells 全部放在一起强行排序。</span></section></li><li><section><span leaf="">UMAP 上看起来有一条线，但这条线主要由 total_counts、pct_counts_mt、细胞周期或批次效应驱动。</span></section></li><li><section><span leaf="">没有明确起始状态，也没有 marker gene、采样时间点或实验设计支持根细胞选择。</span></section></li><li><section><span leaf="">目标过程本身是环形或周期性，例如细胞周期，这类过程不应该简单用单根线性 pseudotime 解释。</span></section></li></ul><p data-tool="mdnice编辑器"><span leaf="">因此，伪时序分析通常不应该在整个对象上直接运行，而是应该先基于细胞注释、marker gene 和研究问题，选出一个生物学上相关的 lineage 或 cell state subset。</span></p><h2 data-cacheurl="" data-remoteid="" data-tool="mdnice编辑器"><span><span leaf="">伪时序的基本思想</span></span></h2><p data-tool="mdnice编辑器"><span leaf="">不同伪时序方法的实现方式不同，但通常包含几个共同步骤：</span></p><pre data-tool="mdnice编辑器"><code><span leaf="">选择细胞群</span><br><span leaf="">-&gt; 构建低维表示或邻居图</span><br><span leaf="">-&gt; 推断轨迹拓扑</span><br><span leaf="">-&gt; 指定起始细胞或起始状态</span><br><span leaf="">-&gt; 计算每个细胞到起点的相对距离</span><br><span leaf="">-&gt; 解释基因表达和细胞状态随 pseudotime 的变化</span><br></code></pre><p data-tool="mdnice编辑器"><span leaf="">其中最关键的是两个问题。</span></p><p data-tool="mdnice编辑器"><span leaf="">第一个问题是轨迹拓扑。也就是细胞状态之间应该被解释成线性、分叉、树状、环状，还是更复杂的图结构。不同工具能处理的拓扑不同。例如 DPT 适合基于扩散距离计算相对顺序，PAGA 更适合先观察 cluster 之间的全局连接结构，Monocle3 会学习 principal graph，Slingshot 则通过 cluster 和 principal curves 推断 lineage。</span></p><p data-tool="mdnice编辑器"><span leaf="">第二个问题是根细胞。多数伪时序方法需要指定起始细胞或起始状态。这个起点不能完全靠算法自动猜，最好来自已知 marker、时间点、实验处理、细胞类型层级或其他独立证据。如果根细胞选错，pseudotime 方向可能整体反过来，甚至得到完全不合理的排序。</span></p><h2 data-cacheurl="" data-remoteid="" data-tool="mdnice编辑器"><span><span leaf="">常见方法怎么选择？</span></span></h2><p data-tool="mdnice编辑器"><span leaf="">当前常见方法可以粗略分成几类。</span></p><h3 data-tool="mdnice编辑器"><span><span leaf="">PAGA</span></span></h3><p data-tool="mdnice编辑器"><span leaf="">PAGA（partition-based graph abstraction）不是单纯的 pseudotime 计算工具，而是用来抽象 cluster 之间的全局拓扑结构。它会在细胞邻居图的基础上，估计不同 cluster 之间是否存在可靠连接。</span></p><p data-tool="mdnice编辑器"><span leaf="">PAGA 适合在正式计算伪时序前回答：</span></p><ul><li><section><span leaf="">当前数据是否像一个连续谱系。</span></section></li><li><section><span leaf="">哪些 cluster 之间可能存在连接。</span></section></li><li><section><span leaf="">是否存在明显分叉结构。</span></section></li><li><section><span leaf="">UMAP 是否扭曲了全局拓扑。</span></section></li></ul><p data-tool="mdnice编辑器"><span leaf="">PAGA 可以和 DPT 搭配使用：先用 PAGA 检查拓扑，再基于合理的 root cell 计算 DPT pseudotime。</span></p><h3 data-tool="mdnice编辑器"><span><span leaf="">DPT</span></span></h3><p data-tool="mdnice编辑器"><span leaf="">DPT（diffusion pseudotime）基于扩散图和随机游走距离，从指定 root cell 出发给每个细胞计算相对伪时序。在 Scanpy 中可以直接使用 </span><code><span leaf="">sc.tl.diffmap()</span></code><span leaf=""> 和 </span><code><span leaf="">sc.tl.dpt()</span></code><span leaf="">。</span></p><p data-tool="mdnice编辑器"><span leaf="">DPT 的优点是容易接入 Scanpy 流程，适合作为 Python 工作流中的基础伪时序方法。它的限制是对 root cell、邻居图和连通性比较敏感；如果数据存在多个分离分支或 disconnected graph，需要谨慎解释。</span></p><p data-tool="mdnice编辑器"><span leaf="">在 Scanpy 中，DPT 更适合用于计算 pseudotime；分支结构和全局拓扑通常更建议结合 PAGA 判断，而不是单独依赖 DPT 的分支检测参数。</span></p><h3 data-tool="mdnice编辑器"><span><span leaf="">Monocle3</span></span></h3><p data-tool="mdnice编辑器"><span leaf="">Monocle3 是 R 生态中常用的轨迹推断工具。它会在低维空间中学习 principal graph，并沿图结构计算 pseudotime。Monocle3 对分叉结构和轨迹可视化支持较好，常用于发育分化问题。</span></p><p data-tool="mdnice编辑器"><span leaf="">需要注意的是，Monocle3 中每个 partition 可以形成独立轨迹。如果某个 partition 没有指定 root node，相关细胞可能得到 infinite pseudotime。因此对多分支或多 partition 数据，需要明确每条轨迹的起点。</span></p><h3 data-tool="mdnice编辑器"><span><span leaf="">Slingshot</span></span></h3><p data-tool="mdnice编辑器"><span leaf="">Slingshot 也是 R 生态中成熟的轨迹推断方法。它通常基于低维坐标和 cluster label，先构建 cluster 间的 minimum spanning tree，再拟合 principal curves，最后给每条 lineage 上的细胞分配 pseudotime。</span></p><p data-tool="mdnice编辑器"><span leaf="">Slingshot 适合已经有较好聚类结果，并且希望在一个或多个 lineage 上平滑拟合轨迹的场景。它允许加入起始 cluster 或终末 cluster 作为先验信息。</span></p><h2 data-cacheurl="" data-remoteid="" data-tool="mdnice编辑器"><span><span 选择和生物学知识验证。</span></p><h3 data-tool="mdnice编辑器"><span><span leaf="">pseudotime 数值能不能解释成真实时间？</span></span></h3><p data-tool="mdnice编辑器"><span leaf="">不能。pseudotime 是相对顺序，不是小时、天或真实发育时间。即使有真实时间点，也只能说
