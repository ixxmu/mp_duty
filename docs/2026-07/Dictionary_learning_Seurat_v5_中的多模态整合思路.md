data-tool="mdnice编辑器"><span leaf="">近年来，单细胞参考图谱越来越多，比如人类骨髓、免疫细胞、脑组织、肺组织等，都已经有了大规模</span><strong><span leaf="">scRNA-seq reference</span></strong><span leaf="">，这些 reference 往往经过人工整理和专家注释，包含比较统一的细胞类型命名体系。如果 query 也是 scRNA-seq，那么可以比较自然地映射到已有 reference 上，做标签迁移、细胞类型注释和跨数据集比较，但像</span><strong><span leaf="">scATAC-seq、CyTOF 这些测得的数据与 scRNA-seq 的特征空间完全不同</span></strong><span leaf="">，不能直接拿基因表达矩阵去对齐。</span></p><p data-tool="mdnice编辑器"><span leaf="">过去有一些方法会把 ATAC 信号转换成</span><strong><span leaf="">gene activity score</span></strong><span leaf="">，也就是把基因 body 或启动子附近的开放性信号当作转录活性的近似。但这种做法隐含了一个比较强的假设：</span><strong><span leaf="">染色质开放就对应基因表达升高</span></strong><span leaf="">。这个假设在很多情况下有道理，但在发育轨迹、细胞命运转换或者调控滞后场景里，并不总是成立。</span></p><p data-tool="mdnice编辑器"><span leaf="">这篇文章提出了一个思路：既然不同模态之间不能直接对齐，那就用一个同时测量两种模态的数据集作为</span><strong><span leaf="">bridge</span></strong><span leaf="">，RNA reference 可以通过</span><strong><span leaf="">bridge integration</span></strong><span leaf=""> 连接到 ATAC query 中，其他模态也可以通过相应的多组学数据集接入同一参考框架，并把这些实现到了</span><strong><span leaf="">Seurat v5</span></strong><span leaf=""> 中。</span></p><figure data-tool="mdnice编辑器"><span><span leaf="">方法框架</span></span></h2><p data-tool="mdnice编辑器"><span leaf="">这里借用了</span><strong><span leaf="">dictionary learning</span></strong><span leaf=""> 的思想。传统 dictionary learning 中，一个样本可以被表示成一组“字典元素”的加权组合。这里作者把</span><strong><span leaf="">multiomic dataset 中的每一个细胞都看作一个 atom</span></strong><span leaf="">，整个 multiomic dataset 就是 dictionary。</span></p><p data-tool="mdnice编辑器"><span leaf="">以 RNA 和 ATAC 的整合为例，流程可以理解为四步：</span></p><p data-tool="mdnice编辑器"><span leaf="">第一步，分别做</span><strong><span leaf="">模态内部的对齐</span></strong><span leaf="">。scRNA-seq reference 和 multiome 的 RNA 部分对齐；scATAC-seq query 和 multiome 的 ATAC 部分对齐，在各自模态内部建立 query/reference 与 bridge 的关系。</span></p><p data-tool="mdnice编辑器"><span leaf="">第二步，构建</span><strong><span leaf="">dictionary representation</span></strong><span leaf="">。对于 RNA reference 中的每个细胞，模型学习它可以由哪些 multiome 细胞加权组合重建；对于 ATAC query 中的每个细胞，模型也学习它可以由哪些 multiome 细胞加权组合重建。因此，尽管 RNA 细胞和 ATAC 细胞的原始特征不同，但最终都被表示成</span><strong><span leaf="">基于同一组 multiome atoms 的权重表示</span></strong><span leaf="">。</span></p><p data-tool="mdnice编辑器"><span leaf="">第三步，降维。如果 multiome 里有几万个细胞，那么 dictionary representation 的维度就会很高。作者使用</span><strong><span leaf="">graph Laplacian eigen decomposition</span></strong><span leaf="">，把这个由 atoms 定义的空间压缩到更低维度，从而提高计算效率。</span></p><p data-tool="mdnice编辑器"><span leaf="">第四步，做</span><strong><span leaf="">跨数据集对齐</span></strong><span leaf="">。当 RNA 和 ATAC 都被转换到同一个</span><strong><span leaf="">atom-defined space</span></strong><span leaf=""> 后，就可以用常规整合方法继续对齐。文章中主要使用 mnnCorrect，但这个框架本身也兼容 Seurat、Harmony、Scanorama、scVI 等方法。</span></p><figure data-tool="mdnice编辑器"><span leaf=""><img data-tool="mdnice编辑器"><span><span leaf="">scATAC-seq 映射到骨髓 RNA reference</span></span></h2><p data-tool="mdnice编辑器"><span leaf="">首先在人骨髓单核细胞数据中验证</span><strong><span leaf="">bridge integration</span></strong><span leaf="">。reference 是一个人骨髓 scRNA-seq reference，包含</span><strong><span leaf="">297,627 个细胞</span></strong><span leaf="">，已经注释了 10 种祖细胞状态和 25 种成熟细胞状态；query 是人骨髓 data-tool="mdnice编辑器"><span><span leaf="">稳健性分析</span></span></h2><p data-tool="mdnice编辑器"><span leaf="">Fig.3a 展示了 bridge 数据规模的影响，先逐步减少 multiome bridge 中的细胞数，再重复映射。结果显示，当某个细胞类型在 bridge 中大约有</span><strong><span leaf="">50 个细胞</span></strong><span leaf=""> 时，预测结果通常已经比较稳定。对于稀有细胞，如果 bridge 中对应细胞太少，预测就更容易波动。</span></p><p data-tool="mdnice编辑器"><span leaf="">Fig.3b–c 比较了</span><strong><span leaf="">bridge integration 和其他多模态整合方法</span></strong><span leaf="">。以 ASDC 这个稀有细胞群为例，比较 multiVI、Cobolt 和 Bridge 的预测结果。在 SIGLEC6 位点的 ATAC 开放信号中，Bridge 预测出的 ASDC 显示出更清楚的细胞类型特异性开放模式，说明它在</span><strong><span leaf="">稀有细胞的高分辨率匹配上更可靠</span></strong><span leaf="">。Fig.3c 则用 ground truth benchmark 定量验证：把同一个 multiome 细胞的 RNA 和 ATAC 拆开，看整合后能不能重新匹配回来。结果显示，Bridge 的</span><strong><span leaf="">Jaccard similarity 更高</span></strong><span leaf="">，说明它更容易把真实对应的跨模态细胞放到一起。</span></p><p data-tool="mdnice编辑器"><span leaf="">Fig.3d–f 使用</span><strong><span data-tool="mdnice编辑器"><span><span leaf="">大规模数据整合</span></span></h2><p data-tool="mdnice编辑器"><span leaf="">除了跨模态映射，文章还提出了</span><strong><span leaf="">atomic sketch integration</span></strong><span leaf="">，用于解决大规模单细胞数据整合的计算问题。随着公开数据不断积累，同一组织往往已经被多个研究、多个平台反复测量。将这些数据统一整合，有助于建立更完整的组织图谱，但</span><strong><span leaf="">全量分析上百万细胞会带来较高的计算负担</span></strong><span leaf="">。</span></p><p data-tool="mdnice编辑器"><span leaf="">它的思路是先从每个数据集中选取一部分代表性细胞作为 atoms，只对这些 atoms 进行整合，再利用 dictionary representation 将整合结果重构回完整数据集。与简单随机抽样不同，作者采用</span><strong><span leaf="">leverage score-based sampling</span></strong><span leaf="">，使稀有细胞也更容易被保留下来，从而在降低计算量的同时尽量维持原始数据结构。</span></p><figure data-tool="mdnice编辑器"><span leaf=""><img leaf="">。每个数据集抽取 5000 个 atoms，最终整合</span><strong><span leaf="">9.5 万个代表性细胞</span></strong><span leaf="">，再重构全部细胞的整合表示。**pulmonary ionocyte 细胞群在整合后数据中仅占约 0.047%**，但仍能被稳定识别，并表达 FOXI1 等经典标记。</span></p><h2 data-tool="mdnice编辑器"><span><strong><span leaf="">总结</span></strong></span></h2><p data-tool="mdnice编辑器"><span leaf="">文章提出了</span><strong><span leaf="">bridge integration</span></strong><span leaf="">，用</span><strong><span leaf="">multiomic dataset 作为不同单细胞模态之间的桥梁</span></strong><span leaf="">。通过把 multiome 中的每个细胞看作 dictionary 中的 atom，RNA reference 和 ATAC、DNA methylation、CyTOF 等 query 数据都可以被转换到同一个由 atoms 定义的空间中。这样一来，已有的</span><strong><span leaf="">scRNA-seq reference 不再只能服务于 RNA 数据</span></strong><span leaf="">，也可以用于其他模态的细胞类型注释和状态解释。</span></p><p data-tool="mdnice编辑器"><span leaf="">同时文章还提出了</span><strong><span leaf="">atomic sketch integration</span></strong><span leaf="">，用代表性细胞作为 atoms，先整合 sketch，再将结果重构回完整数据集。这一策略</span><strong><span leaf="">降低了百万级单细胞数据整合的计算压力</span></strong><span leaf="">，同时尽量保留稀有细胞和整体数据结构。</span></p><p data-tool="mdnice编辑器"><span leaf="">实际应用时需要注意的是，</span><strong><span leaf="">bridge 数据的代表性仍然很关键</span></strong><span leaf="">。如果某些细胞类型或状态在 bridge 中缺失，映射结果就可能受到影响。因此，bridge integration 更适合用于</span><strong><span leaf="">已有较好 reference、并且能够获得具有代表性的 multiomic bridge</span></strong><span leaf=""> 的场景。</span></p></section><p><mp-style-type data-value="10000"></mp-style-type></p></div>
