data-tool="mdnice编辑器"><span><span leaf="">需要注意的点</span></span></h2><ul><li><section><span leaf="">batch_key 的选择非常重要，</span><code><span leaf="">SCVI/SCANVI</span></code><span leaf=""> 会将指定的协变量纳入模型，如果将本应保留的生物学差异错误地作为 nuisance covariate 输入，latent space 的结构就可能受到不必要的影响。如果将本应保留的生物学差异也作为 covariate 输入，就可能出现过校正。</span></section></li><li><section><span leaf="">scArches 流程要求 query 与 reference 使用相同基因集合；如果 query 缺少 reference 基因，就需要补零。论文还指出，随着 reference 与 query 的特征差异增大，性能会下降；因此，reference 构建阶段通常建议保留较大的 HVG 集合，以增加后续 feature overlap 的稳健性。</span></section></li><li><section><span leaf="">scArches 的上限受 reference 质量限制。如果 reference 标签不可靠、覆盖范围不足，或者与 query 在技术平台和生物学背景上差异过大，那么迁移学习和自动注释的可靠性都会受到影响。因此，自动注释结果需要结合 joint embedding、概率分数和 marker genes
