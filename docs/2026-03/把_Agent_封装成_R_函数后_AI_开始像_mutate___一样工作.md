用户本来就习惯把复杂任务拆成函数，再把函数接进向量化、管道化、表格化的工作流里。把 Agent 封装成函数，本质上没有改变这种思路，反而是在延续这种思路：只是过去函数处理的是数值、字符和数据框，现在函数开始处理生物学语义、临床文本和分析上下文。</span></p><p><span leaf="">而 aisdk 在这件事上之所以特别对味，是因为它并不只是提供一个"提问接口"。它把 Agent 和 leaf="">这意味着，在更复杂的项目里，我们不一定非要把一切都做成"单次纯函数调用"。有些场景适合纯函数，比如批量注释、结构化抽取、统一格式；有些场景则更适合带状态的 session，比如连续解读差异分析、逐步润色结果段落、在一个上下文里反复讨论某个 cluster 或病例。aisdk 文档明确说明，ChatSession 可以直接创建，也可以由 agent 创建并继承其 persona 和 tools，同时支持中途切换模型、保存到磁盘、稍后再恢复继续对话。</span></p><p><span leaf="">这就让整件事不止停留在"把 AI 包装成函数"这一招上，而是形成了一个更完整的层次：能批量的地方，函数化；要连续的地方，会话化；需要组合的地方，再进一步工具化和多智能体化。aisdk 首页也把工具调用、memory
