leaf="">为什么要进行数据整合？</span></span></h2><p data-tool="mdnice编辑器"><span leaf="">在单样本流程中，数据通常来自同一个 library。经过质控、归一化、高变基因选择、降维、聚类和注释后，可以得到一个相对完整的分析结果。</span></p><p data-tool="mdnice编辑器"><span leaf="">真实项目中更常见的情况是多个样本共同分析。例如：</span></p><pre data-tool="mdnice编辑器"><code><span leaf="">样本 1：control / donor1 / batch1</span><br><span leaf="">样本 2：control / donor2 / batch1</span><br><span leaf="">样本 3：disease / donor3 / batch2</span><br><span leaf="">样本 4：disease / donor4 / batch2</span><br></code></pre><p data-tool="mdnice编辑器"><span leaf="">如果直接合并所有细胞后做 PCA 和 UMAP，细胞之间的差异可能同时包含两部分信息：</span></p><ol><li><section><span leaf="">真实生物学差异：细胞类型、细胞状态、疾病或处理效应。</span></section></li><li><section><span leaf="">技术差异：建库批次、测序深度、样本处理、10X chemistry 或测序 lane。</span></section></li></ol><p data-tool="mdnice编辑器"><span leaf="">数据整合（data integration）或批次校正（batch correction）的目的，是在共同分析多个样本时减弱不希望保留的技术差异，同时尽量保留真实的生物学结构。</span></p><p data-tool="mdnice编辑器"><span leaf="">需要注意的是，整合不是为了把所有样本强行混在一起。好的整合结果应该是：</span></p><pre data-tool="mdnice编辑器"><code><span leaf="">同一细胞类型内部，不同样本适度混合</span><br><span leaf="">不同细胞类型或真实状态之间，仍然清楚分开</span><br></code></pre><p data-tool="mdnice编辑器"><span leaf="">如果整合后 UMAP 很漂亮，但 marker gene 变得模糊、疾病特异状态消失、稀有细胞群被合并掉，反而可能是过度整合。</span></p><h2 data-cacheurl="" data-remoteid="" data-tool="mdnice编辑器"><span><span leaf=""> 中，同时在 </span><code><span leaf="">adata.obs</span></code><span leaf=""> 中保留每个细胞对应的 </span><code><span leaf="">sample</span></code><span leaf="">、</span><code><span leaf="">donor</span></code><span leaf="">、</span><code><span leaf="">batch</span></code><span leaf=""> 和 </span><code><span leaf="">condition</span></code><span leaf=""> 信息。</span></p><p
