真正的原生python版Monocle2 by 生信编程自修室
------
<div><p data-pm-slice="0 0 []"><span leaf="">单细胞分化轨迹推断领域,Monocle2 是 Trapnell 等在 Nature Biotechnology 2014 提出的经典方法,后续在 Qiu et al. Nature Methods 2017 升级到 v2,核心算法替换为 KDD'15 论文(Mao et al.)提出的 DDRTree —— Discriminative Dimensionality Reduction via learning a Tree。原版 Monocle2 是 R 包, 并且近年已经完全放弃维护，因此用户们要么艰难安装环境,要么转去用 Slingshot、PAGA、Palantir 这些后来者。</span></p><p data-pm-slice="0 0 []"><code><span leaf="">Bio-Babel/Monocle2-python</span></code><span leaf=""> 与 </span><code><span leaf="">Bio-Babel/DDRTree-python</span></code><span leaf=""> 把这两条管线分别从 R 重写成纯 Python:前者负责 Census 计数、density-peak 聚类、</span><code><span leaf="">order_cells</span></code><span leaf="">、BEAM 等高层流程,后者负责真正的 backbone —— 同时学习低维嵌入与主图结构的 reversed graph embedding 算法。本推文以 </span><code><span leaf="">tutorials/V2_1_HSMM.ipynb</span></code><span leaf=""> 为线索,按数据接入 → QC → ordering 基因 → DDRTree → </span><code><span leaf="">order_cells</span></code><span leaf=""> → 可视化 → BEAM 的次序走完 HSMM myoblast 分化的全流程,并在中段把 DDRTree 的目标函数与迭代步逐行拆开。</span></p><h2 align="center"><span leaf="">总览</span></h2><p><span leaf="">Monocle2 解决的问题是把无标签、无时间序列的单细胞表达谱排成一条带分支的连续轨迹,并给每个细胞一个 </span><code><span leaf="">Pseudotime</span></code><span leaf=""> 数值。做这件事的关键不在差异表达,而在</span><strong><span _ddrtree</span></code><span leaf="">),并通过 gold-standard fixture 与 R DDRTree 0.1.6 做数值 parity 测试(</span><code><span leaf="">tests/test_r_parity.py</span></code><span leaf="">、</span><code><span leaf="">Z</span></code><span leaf=""> 通过线性投影 </span><code><span leaf="">W</span></code><span leaf=""> 还能重建 </span><code><span leaf="">X</span></code><span leaf="">。它把"低维空间忠实于原始数据"这件事写进目标。</span></p><p><strong><span leaf="">Laplacian 项 </span><code><span leaf="">λ tr(Y L Yᵀ)</span></code></strong><span leaf=""> —— </span><code><span leaf="">L = diag(B·1) − B</span></code><span leaf=""> 是 MST </span><code><span leaf="">B</span></code><span leaf=""> 上的 graph Laplacian。展开有 </span><code><span leaf="">tr(Y L Yᵀ) = Σ_(i,j)∈edges ‖yᵢ − yⱼ‖²</span></code><span leaf="">,即 MST 所有边长的平方和。最小化这一项等价于让主图节点之间互相靠拢、让 MST 总长度短 —— 但因为 MST 本身是用 </span><code><span leaf="">Y</span></code><span leaf=""> 当前位置上的两两距离构造的,优化时这一项会</span><strong><span leaf="">同时驱动 Y 节点排成尽量短的连通骨架</span></strong><span leaf="">,而骨架在数据真正有分支结构时就会自然分叉,而不是塌成一条线。</span><code><span leaf="">λ</span></code><span leaf=""> 默认取 </span><code><span leaf="">5N</span></code><span leaf=""> 适中时变成"细胞 i 的软分配"。注意整项前面的符号:展开后这一项要被</span><strong><span leaf="">最小化</span></strong><span leaf="">,等价于让每个 </span><code><span leaf="">zᵢ</span></code><span leaf=""> 至少接近某个 </span><code><span 默认。这一项把"细胞被吸附到主图最近的节点"写进目标。</span></p><p><span leaf="">把三项凑在一起:第 1 项让 </span><code><span leaf="">Z</span></code><span leaf=""> 忠实于 </span><code><span leaf="">X</span></code><span leaf="">,第 2
